# 핫딜 웹 수집기 POC 기획 (쿠팡 API 우선 연동)

이 문서는 핫딜 자동 수집 및 할인율 비교 시스템의 개념 검증(POC)을 위해 작성된 기획 및 요구사항 정의서입니다.

---

## 1. POC 목표
- 3대 플랫폼(네이버 쇼핑커넥트, 토스쇼핑 쉐어링크, 쿠팡파트너스) 연동을 최종 목표로 하되, **1단계로 쿠팡 파트너스 골드박스(Goldbox) API**를 통해 상품 목록을 가져오고 DB와 비교하여 조건에 맞는 할인 품목을 리스트로 출력하는 기본 흐름을 검증합니다.

---

## 2. 주요 흐름 (Workflow)

```text
[ 쿠팡 API 호출 ] 
       │ (상품 목록 획득)
       ▼
[ DB 저장 및 가격 비교 ] 
       │ (기존 데이터와 가격/할인율 대조)
       ▼
[ 조건 필터링 ] 
       │ (특정 할인 조건 만족 여부 판단)
       ▼
[ 결과 리스트 출력 (Output) ]
```

1. **API 호출**: 쿠팡 파트너스 골드박스 API(`/v2/providers/affiliate_open_api/apis/openapi/v1/products/goldbox`)를 호출하여 최신의 일일 특가 상품 목록 정보를 수집합니다.
2. **DB 적재 및 비교**: 수집한 상품 정보를 로컬 DB(SQLite)에 저장하고, 이전 수집 데이터와 비교하여 가격 변동을 확인합니다.
3. **조건 필터링**: 할인율, 가격대 등 특정 조건에 부합하는 핫딜 후보를 가려냅니다.
4. **리스트 아웃풋**: 최종 선정된 핫딜 리스트를 확인 가능한 형태(JSON/콘솔/텍스트 파일 등)로 출력합니다.

---

## 3. 플랫폼 확장 계획
- **1단계 (현재)**: 쿠팡 파트너스 API 연동 및 기본 DB 비교/출력 흐름 구현
- **2단계**: 네이버 쇼핑커넥트 및 토스쇼핑 쉐어링크 API 어댑터 추가
- **3단계**: 수집된 핫딜을 시각적으로 확인하고 관리할 수 있는 웹 대시보드(FastAPI + React) 구현

---

## 4. 데이터 모델 (SQLAlchemy / SQLite, PostgreSQL 겸용)

### products (상품 마스터)
- `id` (INTEGER, PK): 상품 일련번호
- `platform` (TEXT): 수집 플랫폼 (`coupang`, `naver`, `toss`)
- `product_id` (TEXT): 플랫폼 내부 상품 ID
- `title` (TEXT): 상품명
- `image_url` (TEXT): 상품 이미지 링크
- `original_url` (TEXT): 원본 상품 링크
- `created_at` (DATETIME): 최초 등록 시간

### price_history (가격 변경 이력)
- `id` (INTEGER, PK): 이력 일련번호
- `product_id` (INTEGER): 상품 FK
- `observed_at` (DATETIME): 수집 일시
- `price` (INTEGER): 수집된 현재 가격
- `discount_rate` (REAL): 플랫폼 표시 할인율 (있는 경우)

---

## 5. POC 검증 기준 (Success Metrics)
- [ ] 쿠팡 API로부터 상품 목록을 정상적으로 파싱해 오는지 확인
- [ ] 파싱한 데이터가 DB에 정상 적재되고, 동일 상품의 이전 가격 기록과 비교가 수행되는지 확인
- [ ] 지정된 조건(예: 할인율 20% 이상 등)을 만족하는 상품만 선별하여 리스트(Output)로 출력되는지 확인
